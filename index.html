inline elements

why these elements are called inline elements

inline elements takes width and height according to their content width and height. inline elements doesn't start on a new line. width and height doesn't work on inline elements.

  • a tag
  • strong
  • em
  • b
  • i
  • q
  • mark

block elements

why these elements are called block elements

block elements takes the whole width. block elements start on a new line. width and height does work on block elements.

  • address
  • aside
  • blockquote
  • details
  • dialog
  • div
  • fieldset

inline-block elements

why these elements are called inline-block elements

inline-block elements doesn't takes the whole width and height. inline-block elements doesn't start on a new line. width and height does work on inline-block elements. Also, with display: inline-block, the top and bottom margins/paddings are respected

  • button
  • select
  • meter
  • progress
  • marquee
  • textarea
  • input

What are Semantic Elements?

A semantic element clearly describes its meaning to both the browser and the developer.
Examples of non-semantic elements: div and span - Tells nothing about its content.
Examples of semantic elements: form, table, and article - Clearly defines its content.
